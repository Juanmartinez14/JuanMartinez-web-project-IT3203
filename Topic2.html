Modern HTTP

HTTP/1.1 Improvements

The creation of HTTP/1.1 improved effciency by allowing constant connection, which reduced the need to constantly open and close connections for more than one request. The HTTP became standarlized in 1997 allowing it to improve scalability and communication between clients and servers.

This version introduced new request methods:

  • PUT - uploads or replaces resources
  • DELETE - removes a resource
  • OPTIONS - checks server capabilities
  • HEAD - retrives metadata only
